
3. VIVI가 사용하는 개인적인 데이터private data를 저장하는 정책을
   정리했습니다. 여기서 용어 한 가지를 정의하고 가야 될 것 같습니다.
   VIVI Private Data:
       1) MTD Partition table: VIVI가 MTD를 다루기 위해 필요한 정보
       2) Parameter table: autboot delay 값, mach type등의 값을
                           가지고 있는 테이블. VIVI가 제대로 동작하기
                           위해서 필수 적인 테이터들.
       3) Linux command line: 리눅스를 부팅 시킬 때 주는 command line
                             문자열.

   이 놈의 디폴트 값은 VIVI내에 하드 코딩되어 있습니다. 하지만 사용자가 사용
   중에 내용을 변경하고 이 값을 영구히 사용하고 싶을 수도 있을 것입니다.
   이 때 문제는 그럼 데이터를 어디에다 저장하느냔데.. 기존의 bootldr을
   'param'이라는 영역을 정의하고 여기에 저장했습니다. 보통 요놈의 위치는
부트로더
   바로 다음에 위치 했었죠.
   회의를 거치면서 우리의 VIVI은 MTD의 제일 마지막 블락에다가 저장하기로
했습니다.
   약간의 유연함flexibility을 주기위해서 제 나름대로 다음과 같이 설계해
봤습니다.

   컴파일 시에 설정에서 사용자는 다음과 같은 세가지 상태의 VIVI를 만들
수 있습니다.

   1) 무조건 VIVI내부의 디폴트 값만 사용한다. 디버깅 할 때나, VIVI
Private data
      의 구조가 바뀌었을 때 필요할 것 같습니다.
   2) 저장은 하되, 디폴트 MTD partition table에 'param'이라고 지정된 영역을
      저장 공간으로 사용합니다. 기존의 bootldr가 비슷합니다.
   3) 회의에서 결정된 바와 같이 MTD의 제일 뒤 한 블락을 저장 공간으로 사
용한다.

   입 맛에 맞는 대로 사용하시면 됩니다.
